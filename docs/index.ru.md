# Быстрый старт

llm-burnwatch отслеживает стоимость каждого вызова LLM/агента в вашем коде,
записывает её в обычный JSONL-файл на вашем собственном диске и находит
аномалии (зацикливание, скачки стоимости, смену модели) — с **нулём
обязательных зависимостей** и **без сетевых вызовов** в ядре. Что именно
покрывает эта гарантия — см. [Модель безопасности](security.ru.md).

## Установка

```bash
pip install llm-burnwatch
```

## Логируем первый вызов

```python
from llm_burnwatch import CostTracker

tracker = CostTracker("calls.jsonl")
tracker.log_call(
    label="summarize",
    model="gpt-4o-mini",
    input_tokens=800,
    output_tokens=150,
)
```

Если вы уже вызываете SDK OpenAI/Anthropic/Gemini/Ollama, используйте
соответствующий адаптер вместо самостоятельного подсчёта токенов — см.
[Подключение к существующему приложению](connecting.ru.md).

## Смотрим, во что это обошлось

```bash
llm-burnwatch report --log-file calls.jsonl
```

## Получаем первый алерт

Реальный трафик для проверки поиска аномалий не нужен — сгенерируйте
синтетический лог с несколькими внедрёнными аномалиями и найдите их:

```bash
llm-burnwatch demo-data --out demo.jsonl
llm-burnwatch detect --log-file demo.jsonl
```

`detect` завершается с кодом `1`, если что-то нашёл (аномалии, нарушения
правил, всплески частоты, сдвиги уровня, алерты бюджета), и `0` — если нет:
удобно встраивать в CI или cron-задачу. Версию с непрерывным слежением,
которая продолжает работать и может отправлять алерты в Slack/Telegram/
вебхук/локальную команду по мере их появления, см. `detect --follow` в
основном README.

## Смотрим как дашборд

```bash
llm-burnwatch dashboard --log-file demo.jsonl --out dashboard.html
open dashboard.html
```

Один самодостаточный HTML-файл — без сервера, без шага сборки, работает
через `file://`. Сортируемые/фильтруемые таблицы и копирование в буфер
обмена реализованы небольшим количеством встроенного vanilla JS (без CDN,
без сетевых вызовов — см. [Модель безопасности](security.ru.md)).
